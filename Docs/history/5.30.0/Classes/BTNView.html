  forceOpacity

Indicates whether the view should ignore a clear color background.

@property (nonatomic, assign, getter=shouldForceOpacity) BOOL forceOpacity

Discussion

Indicates whether the view should ignore a clear color background.

Default is NO. Set to YES to prevent the background color from being set to clearColor by Apple when used as a subview of a highlightable UITableViewCell.

– isVisible

Indicates whether the view is currently visible on the screen.

- (BOOL)isVisible

Discussion

Indicates whether the view is currently visible on the screen.

This method will return YES if the view’s frame, converted to the logical coordinate system of the screen, is contained by the screen bounds and the view and its window have an alpha greater than 0.0 and are not hidden.

Note: There are a couple caveats. 1) The method does not take into account any of its ancestors hidden or alpha values. 2) The method does not take into account whether the view is obscured by another view.
